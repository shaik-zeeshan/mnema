What it does

One on-demand brief, drafted the moment you open the tab — and rendered as the markdown document it exports: ## section headers, - bullets, a filename-style bar, and a persistent action bar. What you see is byte-for-byte what "Copy as Markdown" puts on the clipboard. It reads as a draft that is yours to edit and send, not a report to admire — deliberately unlike Overview's digest lede, which this page previously (wrongly) cloned.

Ownership is visible: AI-written bullets render dim, a bullet you've edited turns bright and is skipped by Regenerate. Every bullet — Yesterday, Today, and Blockers — carries a provenance chip that expands inline to the activities or Context threads it was drawn from. Blockers are a flagged section that never hides: "none detected" is stated, not omitted. Hovering a bullet reveals edit / resummarize-one / remove.

Design provenance: Granola (AI-gray vs. user-black text, per-note source jump), Sunsama (per-item action bar with single-item resummarize), Superhuman (editing freezes regeneration), GitHub Copilot (AI drafts into the editable artifact itself).

How it works

Exactly the digest pattern, one level up. A single structured-extraction call through the shared Reasoning Engine:

  • ai_engine::extract_with_preamble::<StandupSummary>(&engine, &preamble, &prompt) — same call shape digest.rs already uses for DigestNarrative
  • Engine config via the existing single resolver resolve_engine_config(settings, pin, feature_override_model), so it follows the user's provider/model choice and keychain key with zero new provider wiring
  • Result cached per (day, direction) with a freshness floor, mirroring user_context_digests; Regenerate bypasses the floor like Overview's ↻ Re-read

The prompt is built from already-summarized text — activity titles/summaries and digests — never raw frames, so it's cheap and fast even on local models. Generation streams bullets in under the pre-drawn section headers (headers never blank); Regenerate keeps the current brief visible until the replacement lands, and skips any bullet the user has edited. A failed call shows the designed error state — this codebase has already seen extraction come back silently empty once (the DeepSeek tool_choice bug), so "blank card" is an explicitly forbidden outcome here.

Net-new code

  • apps/desktop/src-tauri/src/user_context/standup.rs — small module beside digest.rs: StandupSummary struct (yesterday / today / blockers, each bullet with cited activity/conclusion ids), cache read/write, freshness floor
  • The cache row also stores per-bullet user edits (text + an edited flag) so Regenerate can skip them and Copy exports the edited version — still the same single table
  • One migration in crates/app-infra/migrations for the cache table (a near-copy of 0029_user_context_digests.sql)
  • One #[tauri::command] registered in apps/desktop/src-tauri/src/lib.rs (e.g. get_standup_brief)
  • Frontend: rail entry + tab in InsightsRail.svelte / insights/+page.svelte, one StandupBrief.svelte card reusing the lede visual language

Not built: no background worker (generate-on-open beats generating briefs nobody reads), no new evidence plumbing, no new provider/keychain code, no scheduling.

Data it uses

  • user_context_activities for yesterday + today-so-far (via the same store list_user_context_activities reads)
  • user_context_digests — yesterday's and today's cached narratives as high-level framing
  • Conclusions from the User Context store (crates/app-infra/src/user_context/) — the "open loops" section is where per-belief conclusions about ongoing threads earn their keep

Because every input lives in user_context_* tables, the brief works even after raw retention has purged the underlying frames (ADR 0029) — Monday's standup can still cite Friday. Temporal grounding follows the existing convention: the prompt leads with labeled local time so "yesterday" resolves correctly across midnight and timezones.
