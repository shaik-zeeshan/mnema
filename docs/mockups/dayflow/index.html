Dayflow-style features, inside Mnema

Verdict: we already have the backend — this is mostly presentation. Dayflow's core loop (capture the screen, have an LLM turn it into a chronological journal of activity cards) is exactly what Mnema's User Context worker has been doing since it shipped: every ~30 minutes of capture becomes a titled, summarized, categorized activity row with focus level and evidence frames, and a daily digest narrates the whole day. The four features mocked here are one new Insights sub-tab family (frontend), one small on-demand LLM module cloned from the digest pattern (Standup), and one modal built entirely on existing frame commands (Timelapse). Zero new workers, zero new capture plumbing, one optional table.

Existing raw material: user_context_activities · user_context_digests · user_context_activity_evidence · get_usage_charts · frames + OCR + app/window/url metadata per segment. Each mockup page reproduces the real app chrome (titlebar surface toggle, Insights rail) so it's clear exactly where every feature lives, and carries What-it-does / How-it-works / Net-new-code / Data-it-uses annotations with real file paths.

Decided scope (Jul 3 review): build 01 Journal + 04 Receipt first (the receipt is what makes the journal trustworthy), 02 Blocks as a fast follow inside the same tab, 03 Standup on hold until Journal proves the daily habit.

Grill outcomes (Jul 3, recorded in docs/user-context/CONTEXT.md): Journal = Insights sub-tab (not a third surface); receipt = bounded evidence playback, the one carved exception to "inspection stays on Timeline"; digest lede + stats render in both Overview and Journal (same cached row, shared stat helper — same label ⇒ same computation); day = local midnight, no 4am logical day; pending slot driven by a new coveredUntilMs field on get_user_context_status; card frame counts = frames still on disk. Net: "frontend only" became frontend + three field-level backend touches (watermark field, range-path evidence hydration, is_headline on the wire) — still no new tables, commands, workers, or prompts. Standup's persistence design deliberately deferred until it's scheduled.
