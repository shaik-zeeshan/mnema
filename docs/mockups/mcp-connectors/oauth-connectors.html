MCP connectors

Connect Model Context Protocol servers so chat can use their tools (GitHub, Notion, a filesystem, …). A connector works only when it is both enabled and authorized — two separate things. Some servers sign in with OAuth: click Connect and approve in your browser; others take a pasted token. Either way the secret lives only in the macOS keychain.

Enabled — offered to chat & ✓ authorized holds a token = usable
How connecting works
01 · you click Mnema Connect registers Mnema with the server and opens your browser.
02 · you approve Your browser The provider's own consent screen. This is the gate — nothing is pasted into Mnema.
03 · deep link back Callback mnema://oauth/callback The browser hands the code back through Mnema's own URL scheme — no local server.
04 · token stored Authorized Token saved to the macOS keychain. The row flips to authorized, live.

Refresh happens silently in the background. If it fails, the connector reads needs reconnect — chat reports it as unavailable and you click Reconnect. Mnema never pops a browser on its own.
