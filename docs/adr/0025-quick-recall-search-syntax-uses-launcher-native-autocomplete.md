# Quick Recall search syntax uses a launcher-native autocomplete, not the dashboard dropdown

**Quick Search** in **Quick Recall** gains the same opt-in **Search Query Syntax** the dashboard has ([ADR 0019](0019-search-query-syntax-desugars-into-refinements.md)) — **Field Operator** tokens (`app:`, `source:`, `date:`, `after:`, `before:`) and **Body Match Operator** tokens (quoted phrase, `-term`, `OR`, `term*`). Field operators desugar into visible, removable scope the same way they do on the dashboard, preserving ADR 0019's principle that scope must be **explicit and reversible**. But Quick Recall surfaces that scope through a **launcher-native autocomplete** — inline ghost-text plus a summoned filter picker that replaces the results region — rather than reusing the dashboard's two-tier **Search Operator Suggestion** dropdown.

This is mostly a frontend surfacing change, not a new engine. `search_capture` already runs `parse_search_query` on every query regardless of the caller-supplied `refinements`, so field operators typed into Quick Recall today *already* filter results — but Quick Recall ignores the response's `appliedRefinements`, `residualQuery`, and `parseErrors`. That means scope is applied **invisibly** and a malformed operator silently suppresses all results: the exact "hidden inline scope makes missing results look like search failure" footgun ADR 0019 and ADR 0010 exist to prevent, live in Quick Recall today. This decision stops Quick Recall from ignoring what the engine already returns.

## Considered Options

- **Reuse the dashboard's two-tier dropdown verbatim**: rejected — it collides with Quick Recall's keymap, where `Ctrl+Enter` pivots Search → **Ask AI** and `Arrow`/`Enter` drive the live results list. A dropdown layered over live results would contend for the same keys, and a persistent listbox is heavier than the launcher aesthetic (`Avoid: second dashboard, mini main window`). The launcher-native answer instead lets the value list *replace* the results region only while an operator is open, so the two never fight for arrows.
- **Surface nothing — leave operators as today's invisible inline scope**: rejected — this is the hidden-scope failure mode ADR 0019 already rejected; it is strictly worse in Quick Recall because **Quick Search** spans the user's full retained history, so a silently-dropped `app:` filter or a strict-validation empty result reads as "Mnema found nothing."
- **Bring the dashboard's full persistent chip rail**: rejected in favor of a **lighter launcher chip** — a plain-language pill (`Safari`, `Microphone audio`, `May 1 – May 30`) in a thin band under the input — so the explicit-reversible guarantee is kept without turning the launcher into a mini-dashboard.
- **`/` always opens the filter picker**: rejected — it would block slash-leading searches (`/usr/local/bin`) that [ADR 0007](0007-search-v1-text-search-in-app-db.md) exists to serve. `/` triggers the picker only on an empty input; `Ctrl+F` triggers it anywhere; `Escape` leaves a literal slash.

## Consequences

- The autocomplete is **one value-selection surface reached two ways, plus name-completion ghost-text**. (A) **Inline ghost-text** completes the **Field Operator** *name* (`ap`→`p:`) and, once a value is being typed, the *top matching value* (`app:saf`→`ari` from distinct captured apps); it accepts on `Tab` (any caret position) or `→` (only at end-of-input, `fish`-style) and never consumes `Arrow`/`Enter`. (B) The **Quick Recall Filter Value List** renders in the results region and owns all value *selection*. It appears whenever the caret sits inside an un-committed operator value — reached either by typing the operator directly (`app:`) or through the **Quick Recall Filter Picker** category step (summoned by `/` on empty input or `Ctrl+F` anywhere; selecting a category writes that operator's stub into the input in one step and the picker closes, so the value list is the single navigable list — the door never carries its own) — and **fully replaces live results** while active, owning `Arrow`/`Enter` so exactly one list consumes arrows at any instant. With no value typed it shows the full list and no ghost-text; with a value typed it shows the filtered list plus ghost-text on the top row, the ghost retreating the moment the user arrows into the list. App reuses the distinct-captured-apps query so values are result-bearing; `source:` is three fixed rows; `date:`/`after:`/`before:` show preset/relative rows with a typed-date hint and no calendar or From/To field. The partial operator never reaches the backend (`search_capture` re-runs only on committed-scope changes), so a half-typed value never reads as empty results. Both entries converge on committing a chip and clearing the operator text.
- **Keymap while the value list is up**: `↑/↓` move, `Enter` commits the highlighted value, `Tab`/`→` accept the ghost while it is showing, and `Esc` abandons the in-progress operator (clears the operator token at the caret, keeps the rest of the query; a second `Esc` closes **Quick Recall**). The **Ask AI** pivot is `Ctrl+Enter` only — `Tab` is reserved for ghost-text accept rather than pivoting — and `Ctrl+Enter` is suppressed while the value list is up.
- Field operators desugar into **Quick Recall Filter Chips** — the launcher counterpart of a dashboard **Search Refinement** chip — removable by the pill's `×` or `Backspace` at caret position 0; removal reruns the search with the remaining scope.
- **Result-type narrowing rides the chip**, because Quick Recall has no result-type tabs: an audio `source:` chip collapses Quick Recall to the Audio section (`frameLimit: 0`), while `source:screen` or any `app:` chip collapses it to the Screen section (`audioLimit: 0`). The audio-`source:` + `app:` conflict surfaces as a parse error rather than a section.
- **Parse errors surface as a single inline error line** under the input with a paused-results state, rather than the dashboard's full span-highlighted token treatment; only the first error is shown. The **Ask AI** pivot stays available even while the query has a parse error, since a malformed filter does not block a natural-language question.
- The **Ask AI** pivot inherits the full visible scope. The seed broker search stays **structurally scoped** by the active chips (so the agent's starting evidence matches), while the **question** the agent receives gets a **natural-language scope suffix** derived from the chips — `in Safari`, `from {start} to {end}`, `in microphone audio` — which also renders as the read-only question header. **Body Match Operators** stay verbatim in the residual question text and are applied exactly by the seed FTS.
- Two intentionally **divergent search autocompletes** now coexist — the dashboard's dropdown (a large modal surface) and Quick Recall's ghost-text + picker (a launcher surface). The divergence is justified by surface, not accidental; both desugar field operators into the same backend refinements and honor the explicit-reversible principle.
- **Out of scope / parked**: recent searches (a follow-up of its own — it would add a persisted query-history store that reverses Quick Recall's documented ephemerality and introduces a privacy surface) and saved/watch searches (a separate feature). **Body Match Operators** stay text-only with a small static syntax help affordance near the input rather than picker entries.
